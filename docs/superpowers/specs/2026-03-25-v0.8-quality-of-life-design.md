# v0.8 — Quality of Life Design Spec

## Scope

Items 15-19 from the [1.0 roadmap](../../ROADMAP-1.0.md). Item 20 (per-stream track selection) deferred to its own design cycle.

Implementation order: logging → MKV metadata → pause/resume → verification → post-rip hooks.

---

## 15. Log File Support

### Dependencies

New crates: `log` (facade) + `env_logger` (backend).

### Architecture

New `src/logging.rs` module initializes dual-output logging:
- **stderr** — `Warn` by default, `Info` with `--verbose`, overridable via `RUST_LOG`
- **log file** — always `Debug` level when enabled

Log file path resolution:
1. `--log-file <PATH>` CLI flag (implies logging enabled)
2. `log_to_file = true` in config → auto-log to `~/.local/share/bluback/logs/bluback-YYYY-MM-DD-HHMMSS.log`
3. Default: no log file

Log rotation: keep last 10 files. On init, glob the log directory, sort by mtime, delete oldest beyond 10.

### Config

```toml
log_to_file = false     # default
log_dir = "~/.local/share/bluback/logs/"  # default
```

### CLI

```
--log-file <PATH>    Write log to this file (overrides config)
```

### What Gets Logged

- Disc detection and volume label parsing
- AACS preflight results
- Playlist scan results (count, durations)
- Stream selection decisions
- Remux start/stop per file (not every 100ms progress tick)
- Chapter injection count
- Verification results
- Hook execution and output
- Errors and warnings (all existing `eprintln!("Warning: ...")` become `warn!()`)

### Integration

`logging::init()` called in `main.rs` after config load, before AACS preflight. FFmpeg's custom log callback in `probe.rs` forwards to `log::debug!()`.

## 16. Pause/Resume During Ripping

### Mechanism

Add `pause: Arc<AtomicBool>` to `RemuxOptions` alongside existing `cancel`.

In the remux packet loop (after the cancel check), add:

```rust
while options.pause.load(Ordering::Relaxed) {
    if options.cancel.load(Ordering::Relaxed) {
        break;
    }
    std::thread::sleep(Duration::from_millis(100));
}
```

Cancel always takes priority — a paused rip can still be aborted.

### TUI Integration

- `p` key in `dashboard::handle_input` toggles `app.rip.pause`
- Title bar shows `"Ripping: X/Y complete  |  PAUSED"` when paused (replaces stats line)
- Key hints: `[p] Resume` when paused, `[p] Pause` when running
- Progress bar and ETA freeze naturally (no progress callbacks fire while sleeping)

### CLI Mode

Not exposed. TUI only. CLI users have `Ctrl+Z` (SIGTSTP) for shell job control.

### Changes

- `RemuxOptions`: add `pause: Arc<AtomicBool>`
- `workflow::prepare_remux_options()`: takes additional `pause` parameter
- `RipState`: stores `pause: Arc<AtomicBool>` alongside `cancel`
- `dashboard::start_next_job()`: passes pause flag to `prepare_remux_options()`

## 17. MKV Metadata Embedding

### Approach

Set metadata on the output `AVFormatContext` before `write_header()` — same location where chapters are already injected.

### New Field on `RemuxOptions`

```rust
pub metadata: HashMap<String, String>,
```

### Auto-Populated Metadata

| Mode | Key | Example |
|------|-----|---------|
| TV | `title` | `S01E03 - The Train Job` |
| TV | `show` | `Firefly` |
| TV | `season_number` | `1` |
| TV | `episode_id` | `3` |
| TV multi-ep | `title` | `S01E03-E04 - Episode Name` |
| TV multi-ep | `episode_id` | `3` (first episode) |
| Movie | `title` | `Serenity (2005)` |

`title` is the only field with broad real-world value (VLC, mpv, Jellyfin, Plex all read it). The others are written as free extras — they don't hurt and may be useful to some players.

### Custom User Metadata

Config (TOML inline table):

```toml
[metadata]
studio = "Fox"
source = "Blu-ray"
```

CLI (repeatable flag):

```
--metadata key=value --metadata source=Blu-ray
```

Merge order (last wins): auto-populated → config → CLI. User keys override auto keys on conflict (e.g., `--metadata title="Custom"` wins).

### Implementation

In `remux()`, after `inject_chapters()` and before `write_header_with()`, iterate the metadata map and set on `octx.metadata_mut()`.

`workflow::prepare_remux_options()` builds auto metadata, merges config metadata, then CLI metadata.

Config type: `metadata: Option<HashMap<String, String>>` (added to `Config` struct).

No config toggle for auto-metadata — always written. Custom metadata is purely additive.

## 18. Post-Rip Hooks

### Config

```toml
post_rip_command = "notify-send 'Rip complete' '{file}'"
```

### CLI

```
--post-rip-command <CMD>    Command to run after each file (overrides config)
```

### Template Variables

| Variable | Example | Notes |
|----------|---------|-------|
| `{file}` | `/output/S01E03_Episode.mkv` | Full absolute path |
| `{filename}` | `S01E03_Episode.mkv` | Basename only |
| `{title}` | `The Train Job` | Episode or movie title |
| `{show}` | `Firefly` | Series name (empty for movies) |
| `{season}` | `1` | Season number |
| `{episode}` | `3` | Episode number |
| `{playlist}` | `00801` | Playlist number |

### Execution

- Template variables expanded using existing `util` template rendering
- Runs via `std::process::Command` with `sh -c` (supports pipes, redirection)
- **Non-fatal:** hook failure logs a warning but does not mark the rip as failed
- Stdout/stderr captured and logged at `info`/`warn` level
- No timeout

### Integration

Shared function in `workflow.rs` called by both TUI and CLI:

```rust
pub fn run_post_rip_hook(command_template: &str, vars: &HashMap<&str, String>) -> Result<()>
```

- **TUI:** in `poll_active_job()`, after `Disconnected` → `Done`, spawn hook in fire-and-forget thread
- **CLI:** run hook synchronously after each file completes

### Sequencing

Per-file only (no per-session hook). Runs after verification, before starting the next job.

## 19. Rip Verification

### What It Checks

1. **Duration match** — probe output MKV duration, compare against source playlist expected duration. Pass if within 2-second tolerance (accounts for rounding in chapter boundaries and codec frame alignment).
2. **Stream count** — verify output has expected number of streams (based on what was selected for remux). Catches silently dropped streams.

### New Module: `src/verify.rs`

```rust
pub struct VerifyResult {
    pub passed: bool,
    pub expected_duration_secs: u32,
    pub actual_duration_secs: u32,
    pub expected_streams: usize,
    pub actual_streams: usize,
    pub warnings: Vec<String>,
}

pub fn verify_output(
    path: &Path,
    expected_duration_secs: u32,
    expected_streams: usize,
) -> Result<VerifyResult>
```

Implementation uses `ffmpeg_the_third` to open the output file, reads duration from `AVFormatContext`, counts streams. Header inspection only — fast (<1 second).

### Config

```toml
verify = true    # default
```

### CLI

```
--no-verify    Skip post-rip verification
```

### Integration

Runs automatically after every successful remux, before the post-rip hook.

- **TUI:** called in `poll_active_job()` after `Disconnected` → `Done`. On verification failure, file is kept, warning logged, `!` indicator shown next to "Completed" in job table.
- **CLI:** prints verification result per file. Warning on mismatch.

No auto-delete of files that fail verification. User decides.

### Post-Remux Sequencing

1. Remux completes → `Done(file_size)`
2. Verify output (if enabled) → log result, update status indicator
3. Run post-rip hook (if configured)
4. Start next job
